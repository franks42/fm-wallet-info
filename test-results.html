üêõIssue Discovered

While Playwright and the testing infrastructure work perfectly, we discovered a bug in the application's ClojureScript code that prevents the HASH price from loading.

Error Details

Scittle error: Could not resolve symbol: r
Location: src/fm_wallet.cljs:7:43
Phase: analysis

Context:
03    (:require [reagent.core :as r]
04              [reagent.dom :as rdom]))
05
06  (js/console.log "üì¶ CLJS: Namespace fm-wallet loading...")
07  (js/console.log "üì¶ CLJS: Reagent core:", (if r "‚úÖ Found" "‚ùå Missing"))
                                              ^--- Could not resolve symbol: r
08  (js/console.log "üì¶ CLJS: Reagent dom:", (if rdom "‚úÖ Found" "‚ùå Missing"))

Root Cause

The code attempts to use namespace aliases (r and rdom) immediately after the :require statement. However, Scittle loads namespaces asynchronously, so these symbols aren't available yet when lines 7-8 execute.

Impact

  • ClojureScript execution stops at line 7
  • The init() function never runs
  • No API call is made to Figure Markets
  • App remains stuck in "Loading..." state

Fix Required

Remove or relocate the debug console.log statements on lines 7-8 that reference namespace aliases before they're actually used in the code.

üîßEnvironment Workarounds

Two critical fixes were required to get Playwright working in the Claude Code environment:

1. Custom TMPDIR (Browser Crash Fix)

  • Problem: /tmp directory permission issues
  • Symptom: Browser crashes immediately
  • Solution: Export custom TMPDIR=/home/user/PROJECT/.tmp

2. Local CDN Resources (Network Fix)

  • Problem: Browser cannot access external CDNs
  • Symptom: ERR_TUNNEL_CONNECTION_FAILED
  • Solution: Download and serve CDN files locally
